# Feature Reference

> Complete feature breakdown for SecureYeoman. For installation and usage, see the [Getting Started Guide](guides/getting-started/getting-started.md).

| Category | Features |
|----------|----------|
| **Security** | RBAC (Admin/Operator/Auditor/Viewer), JWT + API key auth, mTLS, AES-256-GCM encryption at rest, sandboxed execution (Landlock/macOS sandbox), rate limiting (per-user, per-IP, global), HTTP security headers (HSTS, CSP, X-Frame-Options), CORS policy enforcement; **ToolOutputScanner** — 18-pattern credential leak detector (OpenAI/Anthropic keys, GitHub PATs, AWS credentials, PEM private keys, DB connection strings, bearer tokens, JWTs, Slack/Stripe/Discord tokens, generic API key assignments) scans every LLM response before delivery and replaces matches with `[REDACTED:<type>]`; **Skill Trust Tiers** — community skills (`source: 'community'`) are restricted to read-only tool access (26 name-prefix allow-list), preventing shell execution, file writes, and arbitrary HTTP from unvetted skill instructions; **Outbound Credential Proxy** — `CredentialProxy` runs in the parent process and injects `Authorization` headers for known API hosts; sandboxed children receive only `http_proxy=http://127.0.0.1:PORT`, never the raw secret; HTTPS CONNECT tunnels enforce a per-sandbox hostname allowlist (ADR 099) |
| **Observability** | Cryptographic audit trails (HMAC-SHA256 chain), Prometheus metrics, Grafana dashboards, structured JSONL log rotation, audit retention enforcement, audit export |
| **AI Integration** | Anthropic Claude, OpenAI GPT, Google Gemini, Ollama, LM Studio, LocalAI (local), OpenCode Zen, DeepSeek, Mistral, x.ai Grok, Letta (stateful agent platform with persistent memory); automatic fallback chains on rate limits/outages; dynamic model discovery |
| **Dashboard** | React + Vite + Tailwind; real-time WebSocket updates (channel-based RBAC); overview with stat cards (tasks, heartbeat, audit, memory) and services status panel (core, Postgres, audit chain, MCP); system flow graph (ReactFlow) with live connection edges; task history, security events, resource monitor, personality editor (Brain section shows associated skills with direct edit navigation), skills manager, code editor (Monaco), notification & retention settings; **WebGL graph visualization** (Sigma.js + graphology) with pluggable layout algorithms — ForceAtlas2 for hub-and-spoke peer networks, Dagre hierarchical layout for delegation trees and DAGs; **rich chat rendering** — three-phase assistant messages (Thinking → Tools used → Response) with `ThinkingBlock` collapsible, grey tool-call badges, and primary sparkle creation cards; tool-call badges and thinking content persist across page reloads (saved to DB); full Markdown with syntax-highlighted code (Prism, dark/light theme-aware), interactive Mermaid diagrams, KaTeX math expressions, GitHub-style alert callouts, task list checkboxes, and styled tables; **real-time collaborative editing** — personality system prompts and skill instructions use Yjs CRDT over a binary `/ws/collab` WebSocket; **presence indicators** — `PresenceBanner` shows who else is editing the same field in real time (colored dots + name label); **Group Chat view** (`/group-chat`) — unified three-pane channel list + message thread + reply box aggregating all connected integrations, with personality tracking per message; **Cross-Integration Routing Rules** — visual rule builder embedded in the Connections page (Routing Rules tab) for priority-ordered rules that forward, reply, override personality, or notify webhooks across integrations |
| **Agent Architecture** | Soul (identity/archetypes/personality), Spirit (passions/inspirations/pains), Brain (memory/knowledge/skills with decay & pruning, vector semantic search via FAISS/Qdrant/ChromaDB, LLM-powered memory consolidation), Body (heartbeat/vital signs/screen capture, per-personality capabilities: vision, auditory, vocalization, limb movement, haptic; **per-personality active hours** — configurable rest schedule suppresses heartbeat checks and proactive triggers outside the defined window); **Multi-active personalities** — multiple personalities run simultaneously; `is_active` is non-exclusive; `is_default` is the exclusive new-chat/dashboard default; per-personality enable/disable/set-default endpoints; `Active`, `Default` (star), `Online` (pulsing), and `Preset` badges in the PersonalityEditor; **Archetype protection** — built-in preset personalities (`is_archetype = true`) are permanently protected from deletion; **Personality Presets** — built-in selectable templates (`FRIDAY` general assistant, `T.Ron` security watchdog) instantiated via `GET /api/v1/soul/personalities/presets` + `POST .../presets/:id/instantiate` |
| **Cognitive Architecture** | Vector semantic memory (local SentenceTransformers + OpenAI/Gemini API embeddings), FAISS, Qdrant, and ChromaDB vector backends, LLM-powered memory consolidation with on-save dedup and scheduled deep analysis, 3-tier progressive history compression (message → topic → bulk) with AI summarization; **Hybrid FTS + RRF search** — `tsvector` GIN index on `brain.memories` + `brain.knowledge` merged with pgvector via Reciprocal Rank Fusion, improving recall for exact terms and named entities; **Content-chunked indexing** — large documents split into 800-token overlapping chunks with independent FTS + vector indexes; **Proactive context compaction** — token-usage estimated before each LLM call, older turns summarised at 80% context-window fill to prevent overflow; **Self-repairing task loop** — `TaskLoop` detects stuck agents (timeout or repeated tool calls) and injects diagnostic recovery prompts |
| **Extensions** | 38 lifecycle hook points (observe/transform/veto semantics), TypeScript plugin modules with filesystem discovery, EventEmitter integration, outbound webhook dispatch with HMAC signing, hot-reload support |
| **Code Execution** | Sandboxed code execution (Python, Node.js, shell) within Landlock/seccomp sandbox, persistent sessions, streaming output via WebSocket, approval policies (manual/auto/session-trust), streaming secrets filter, full audit trail |
| **A2A Protocol** | Agent-to-Agent cross-instance delegation via E2E encrypted messaging, peer discovery (mDNS/DNS-SD/static), capability negotiation, trust progression (untrusted/verified/trusted), remote delegation in unified delegation tree |
| **Multi-Agent Architecture** | Sub-agent delegation system with role-based profiles (researcher, coder, analyst, reviewer, summarizer); Agent Swarms with named templates and three strategies — `sequential` (context-chaining pipeline), `parallel` (`Promise.all` + optional coordinator synthesis), `dynamic` (coordinator-driven, uses `delegate_task` internally); `create_swarm` MCP tool; 4 built-in templates; dashboard Swarms tab; **Dynamic Tool Creation** — agents can generate and register new tools at runtime (Agent Zero-style), gated by `allowDynamicTools` security policy with `sandboxDynamicTools` isolation; **Extensible sub-agent types** — `llm` (agentic loop), `binary` (spawn external process via JSON stdin/stdout, zero token cost, gated by `allowBinaryAgents`), `mcp-bridge` (call MCP tool directly with Mustache template, zero token cost); **Intelligent Model Routing** — heuristic task profiler selects the cheapest appropriate model per delegation (fast tier for summarise/classify/extract, capable tier for code/reason/plan), cost-aware swarm scheduling injects model overrides per role, `POST /api/v1/model/estimate-cost` provides pre-execution cost estimates with cheaper-alternative suggestions |
| **Integrations** | 31 platforms — Telegram (inline keyboards, document attachments), Discord (threads, modals, slash command registration via REST), Slack (Block Kit actions, modal dialogs, Workflow Builder steps), GitHub (PR review automation, issue auto-labeling, code search triggers), GitLab, Google Chat, Gmail, Email (IMAP/SMTP), Google Calendar, Notion, Jira, AWS, Azure DevOps, Linear, Airtable, DingTalk, LINE, QQ, Signal, WhatsApp, Microsoft Teams, Twitter/X, Spotify, Stripe, YouTube, Zapier, Figma, Todoist, iMessage, CLI, Generic Webhook — plugin architecture with unified message routing |
| **MCP Protocol** | Standalone `@secureyeoman/mcp` service (59+ tools including web scraping, search, browser automation placeholders; 7 resources, 4 prompts); SSRF-protected web tools; health monitoring for external servers; AES-256-GCM encrypted credential storage; streamable HTTP, SSE, and stdio transports; feature toggles with dashboard UI; one-click pre-built integrations for Bright Data, Exa, E2B, Supabase, Figma, Stripe, Zapier, Linear, Meilisearch, Qdrant, Device Control (camera/printer/audio/screen), ElevenLabs (voice cloning), Home Assistant, Coolify (MetaMCP), and **QuickBooks Online**; TTS/STT provider routing — OpenAI or Voicebox local Qwen3-TTS/Whisper via `TTS_PROVIDER` / `STT_PROVIDER` env vars; **Kali Security Toolkit** (`sec_*` tools: nmap, gobuster, ffuf, sqlmap, nikto, nuclei, whatweb, wpscan, hashcat, john, theHarvester, dig, whois, shodan) gated by `MCP_EXPOSE_SECURITY_TOOLS` with scope enforcement; **Agnostic QA Bridge** (10 `agnostic_*` tools) — REST bridge (`agnostic_submit_qa`, `agnostic_task_status`, status/session/reporting tools) plus `agnostic_delegate_a2a` for structured A2A protocol delegation; gated by `MCP_EXPOSE_AGNOSTIC_TOOLS`; stack auto-starts via `AGNOSTIC_AUTO_START=true`; **QuickBooks Online** (59 `qbo_*` tools) — full CRUD for 11 accounting entities (Account, Bill, BillPayment, Customer, Employee, Estimate, Invoice, Item, JournalEntry, Purchase, Vendor), company info, and P&L / Balance Sheet reports; OAuth 2.0 refresh-token auth; gated by `MCP_EXPOSE_QUICKBOOKS_TOOLS` |
| **Marketplace** | Skill discovery, search, install/uninstall (syncs with Brain skills), publish; **Skill Import/Export** — export AI-learned skills as portable `.skill.json` files; import skills from JSON with `$schema: 'sy-skill/1'` validation; **Community Skills** — git-fetch sync (`allowCommunityGitFetch` policy pulls directly from [`secureyeoman-community-repo`](https://github.com/MacCracken/secureyeoman-community-repo) or a custom `COMMUNITY_GIT_URL`) with automatic prune of removed skills; local-path fallback; source tracking (`builtin` / `community` / `published`) |
| **Team Collaboration** | Multi-user foundation (`auth.users`); workspaces with isolation, member management, workspace-scoped RBAC; **SSO/OIDC** — Okta, Azure AD, Auth0 and any standards-compliant OIDC issuer via `openid-client` v6; PKCE flow; JIT user provisioning; per-workspace IDP binding; **CRDT collaborative editing** — Yjs Y.Text over `/ws/collab/:docId` (personality/skill scoped); server-resolved presence identities (display name from soul users table); DB-backed Y.Doc state (`soul.collab_docs`) survives server restarts |
| **Data Loss Prevention** | Content classification (PII regex, keyword dictionary, custom patterns, 4-tier levels), DLP policy engine (block/warn/log actions), egress scanning with anomaly detection (z-score), data retention policies (per-content-type, classification-aware, automated purge), invisible watermarking (3 algorithms: unicode-steganography, whitespace encoding, homoglyph substitution), 22 REST endpoints, 6 MCP tools, dashboard widget |
| **Supply Chain Security** | SBOM generation (CycloneDX), SLSA Level 3 provenance (Sigstore), signed releases (cosign on binary/Docker/Helm), compliance mapping (NIST 800-53, SOC 2, ISO 27001, HIPAA, EU AI Act), dependency provenance tracking (Socket.dev), reproducible Docker builds (pinned digests) |
| **Multi-Region HA** | Read replica connection pools (round-robin), cross-cluster federation (encrypted peer sync), configurable failover, `getReadPool()` for read-only queries, replica health monitoring |
| **Agent Evaluation** | Scenario-based eval harness, test suites with expected outcomes, multi-metric scoring (tool accuracy, arg match, semantic similarity), pass/fail thresholds, REST API + MCP tools |
| **Reports & Analytics** | Audit report generator (JSON/HTML/CSV), compliance report generator (cross-reference audit/DLP/classification), cost optimization with routing suggestions and forecasting, A/B testing framework, event subscription system (14 event types, HMAC-signed webhook delivery, retry with backoff) |
| **Voice** | Push-to-talk (Ctrl+Shift+V), browser-native speech recognition & synthesis, voice overlay |
| **Deployment** | **Single binary** (Bun compile, ~123 MB incl. runtime, no runtime deps) for Linux x64/arm64, macOS arm64, and Windows x64; Tier 2 SQLite `lite` binary for edge/embedded (Linux and Windows); Docker image (binary-based); Kubernetes Helm chart (EKS/GKE/AKS), GHCR image registry, HPA autoscaling, PodDisruptionBudgets, NetworkPolicies, ExternalSecret CRD support |
| **CLI** | 56 commands with `--json` scripting output and shell completions (bash/zsh/fish). **Core**: `start`, `health`, `status`, `config`, `init`, `migrate`; **AI**: `model`, `provider`, `chat`, `strategy`; **Agents**: `agents`, `crew`/`team`, `a2a`, `world`; **Security**: `policy`, `sandbox`, `dlp`, `guardrail`, `security` (Kali toolkit), `tee`/`confidential`, `athi`/`threat`, `risk`; **Workflows**: `workflow`/`wf`; **Training**: `training`/`train`, `federated`/`fl`; **Knowledge**: `memory`/`mem`, `knowledge`/`kb`; **Observability**: `observe`/`obs`, `alert`, `audit`, `replay`; **Governance**: `pac`/`policy-as-code`, `iac`, `role`, `tenant`, `license`/`lic`; **Integrations**: `integration`/`int`, `plugin`, `agnostic`/`ag`, `mcp-server`, `mcp-quickbooks`/`mcp-qbo`, `skill`/`marketplace`; **Development**: `repl`/`shell`, `tui`/`dashboard`, `execute`, `browser`/`br`, `scraper`/`sc`, `multimodal`/`mm`, `sbom`/`bom`, `verify`; **Utility**: `alias`, `completion`, `extension`, `personality`/`pers`, `chat`, `help`. See [CLI Reference](guides/cli-reference.md) |
| **Workflows** | DAG-based workflow orchestration engine — 14 step types (`agent`, `tool`, `mcp`, `condition`, `transform`, `resource`, `webhook`, `subworkflow`, `swarm`, `data_curation`, `training_job`, `evaluation`, `conditional_deploy`, `human_approval`); Mustache-style `{{steps.id.output}}` data-flow templates; topological sort (Kahn's algorithm) with cycle detection; per-step retry policies, 4 error modes (fail/continue/skip/fallback); 5 trigger types (manual, schedule, event, webhook, skill); per-step output schema validation; autonomy levels (L2 human-on-the-loop, L3 human-in-the-loop approval gate); ReactFlow visual DAG builder with draggable step-type palette and live config panel; run timeline page with step-level status, duration, and collapsible input/output JSON; 6 built-in templates (Research Report Pipeline, Code Review + Webhook, Parallel Intelligence Gather, Distill and Eval, Finetune and Deploy, DPO Loop); REST API (9 endpoints); 5 MCP tools (`workflow_list`, `workflow_get`, `workflow_run`, `workflow_run_status`, `workflow_cancel`); dedicated `/workflows` dashboard page + `/tasks` page powered by `TaskHistory` |
| **AI Training Pipeline** | Conversation dataset export (ShareGPT / instruction / raw formats) via `POST /api/v1/training/export`; **Distillation** — `DistillationManager` curates teacher-student conversation pairs, exports JSONL training data, tracks job status; **LoRA Fine-tuning** — `FinetuneManager` + Unsloth Docker sidecar (`Dockerfile.unsloth-trainer`) executes LoRA training, streams logs, registers adapter with Ollama; **ML Pipeline Orchestration** (Phase 73) — `DataCurationManager`, `EvaluationManager`, `PipelineApprovalManager`, `PipelineLineageStorage` wired to the workflow engine's 5 ML step types; 3 built-in ML pipeline templates; human approval gates with timeout; pipeline lineage tracking (`training.pipeline_lineage`); REST API at `/api/v1/training/*`; TrainingTab in DeveloperPage |
| **Knowledge Base & RAG** | Document-oriented knowledge platform — `brain.documents` tracking table + `brain.knowledge_query_log`; `DocumentManager` ingest pipeline: extract text (PDF via `pdf-parse`, HTML tag-stripping, Markdown, plain text) → `chunk()` → `brainManager.learn()` per chunk → vector-index; **Connectors**: URL crawl, GitHub Wiki (`.md` files via GitHub contents API), paste-text; file upload endpoint (multipart, 20 MB); `deleteDocument` removes all associated chunks via source prefix; **Knowledge Health** — query log analytics (24h query count, avg top score, low-coverage queries, format breakdown); 4 MCP tools: `kb_search`, `kb_add_document`, `kb_list_documents`, `kb_delete_document`; Documents/Connectors/Health tab in VectorMemoryExplorerPage |
| **Enterprise** | **Multi-tenancy** — PostgreSQL RLS partitioning with `tenant_id` on 9 core tables, `TenantManager` CRUD, `withTenantContext()` session variable wiring, admin routes `/api/v1/admin/tenants/*`; **SAML 2.0** — `SamlAdapter` wraps `node-saml` (lazy import), `SsoManager` handles both OIDC and SAML, SP metadata endpoint, ACS route, group-to-role mapping via `groupRoleMap`; **Backup & DR** — `BackupManager` spawns `pg_dump`/`pg_restore`, `admin.backups` table, 6 REST endpoints, download stream, restore confirmation string; **Audit Export** — `POST /api/v1/audit/export` streams JSONL/CSV/syslog RFC 5424; **Multi-Instance Federation** — `FederationManager` with SSRF-guarded peer registration, AES-256-GCM shared-secret encryption, health cycle, federated knowledge search, personality bundle export/import (`.syi` files, passphrase-encrypted); **API Gateway Mode** — gateway API keys with RPM/TPD enforcement, per-request usage tracking, latency analytics (p50/p95), CSV export; **Prompt Security** — jailbreak scoring (weighted pattern severity, `jailbreakAction`: block/warn/audit_only), system-prompt leak detection (trigram overlap), abuse detection (topic pivot Jaccard, tool anomaly, blocked-retry cool-down) |
| **Development** | TypeScript strict mode, ~21,000 tests across 900+ files, CI/CD pipeline (lint/typecheck/test/build/security audit/docker-push/helm-lint); **Storybook** component development environment integrated into the Developers section (gated by `allowStorybook` security policy); single binary build (`bun build --compile`) for Linux x64/arm64, macOS arm64, Windows x64; `--dev` fast single-platform build mode |
