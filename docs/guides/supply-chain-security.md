# Supply Chain Security Guide

SecureYeoman provides built-in supply chain security tools for SBOM generation, release verification, compliance mapping, and dependency provenance tracking.

## SBOM Generation

Generate a CycloneDX 1.5 Software Bill of Materials:

```bash
# Generate SBOM to stdout
secureyeoman sbom generate

# Write to file
secureyeoman sbom generate --output sbom.cdx.json

# Include dev dependencies
secureyeoman sbom generate --include-dev

# Specify custom root directory
secureyeoman sbom generate --dir /path/to/project
```

The SBOM is automatically generated and attached to every GitHub Release.

## Release Verification

Verify the integrity of downloaded binaries:

```bash
# Download binary + checksums
curl -LO https://github.com/.../secureyeoman-linux-x64
curl -LO https://github.com/.../SHA256SUMS

# Verify checksum
secureyeoman verify secureyeoman-linux-x64

# Verify with cosign signature (requires cosign CLI)
secureyeoman verify secureyeoman-linux-x64 --cosign

# JSON output for automation
secureyeoman verify secureyeoman-linux-x64 --json
```

### Manual checksum verification

```bash
sha256sum -c SHA256SUMS --ignore-missing
```

### Cosign signature verification

```bash
cosign verify-blob \
  --signature secureyeoman-linux-x64.sig \
  --certificate secureyeoman-linux-x64.cert \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  secureyeoman-linux-x64
```

## Compliance Framework Mapping

View how SecureYeoman features map to regulatory compliance controls:

```bash
# Show coverage summary for all frameworks
secureyeoman sbom compliance

# Detailed mapping for a specific framework
secureyeoman sbom compliance --framework nist-800-53
secureyeoman sbom compliance --framework soc2
secureyeoman sbom compliance --framework iso27001
secureyeoman sbom compliance --framework hipaa
secureyeoman sbom compliance --framework eu-ai-act

# JSON output for integration with compliance tools
secureyeoman sbom compliance --json
```

### Supported Frameworks

| Framework | Controls Mapped | Description |
|-----------|----------------|-------------|
| NIST SP 800-53 Rev 5 | 24 | US federal information security standard |
| SOC 2 Type II | 14 | Trust Services Criteria (AICPA) |
| ISO 27001:2022 | 14 | International information security management |
| HIPAA Security Rule | 13 | US healthcare data protection |
| EU AI Act | 9 | European AI regulation |

Each mapping entry includes:
- **Control ID**: The framework-specific control identifier
- **Feature**: The SecureYeoman capability that satisfies the control
- **Evidence**: Source file or module providing the evidence
- **Status**: Implementation status (implemented/partial/planned)

## Dependency Provenance Tracking

Monitor your dependency tree for supply chain risks:

```bash
# Check for changes since baseline
secureyeoman sbom deps

# Update baseline after accepting changes
secureyeoman sbom deps baseline

# JSON output for CI integration
secureyeoman sbom deps --json
```

### Risk Levels

| Level | Category | Description |
|-------|----------|-------------|
| CRITICAL | Registry change | Package resolved from different registry |
| CRITICAL | Integrity mismatch | Hash changed without version bump |
| HIGH | Bulk new deps | >10 new production dependencies at once |
| MEDIUM | New dependency | New production dependency added |
| MEDIUM | Major version bump | Semver major version change |
| INFO | New dev dependency | New development dependency |

### CI Integration

Add dependency tracking to your CI pipeline:

```yaml
- name: Check dependency provenance
  run: secureyeoman sbom deps --json
  # Exits non-zero on critical/high alerts
```
